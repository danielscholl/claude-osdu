# Merge Request Description Prompt

This is the reference for generating MR descriptions from branch context. Generate the
description directly based on the commit log and diff stats.

## Prompt

Generate a merge/pull request description in markdown from the commit log and diff stats
provided below.

Structure:
1. **Summary** — 1-3 sentences explaining *what* changed and *why*. Focus on the motivation,
   not just listing files. What problem does this solve or what capability does it add?
2. **Changes** — A bullet list of the notable changes, grouped logically. Use sub-bullets for
   related items. Don't list every file — summarize by area or purpose.
3. **Notes** (optional) — Only include if there are things reviewers should pay attention to:
   breaking changes, migration steps, dependencies, or things that need manual verification.

Rules:
- Write for a reviewer who knows the codebase but hasn't seen this branch
- Lead with the "why", not the "what"
- Be specific — "add Redis cluster deployment" not "update platform"
- Keep it concise — the full diff is available in the MR itself
- No attribution lines, no "Generated by" footers
- Output ONLY the markdown body — no fences, no preamble, no title

Example output:

## Summary

Standardize skill directory naming so infrastructure skills group under a common `iac-` prefix,
making them easier to discover and maintain.

## Changes

- **Skill renames**: `terraform` → `iac-terraform`, `infrastructure-debugging` → `iac-debug`,
  `verify-infrastructure` → `iac-verify`, `obsidian-vault` → `brain`
- **New `send` skill**: automates the commit → push → MR workflow using git and glab
- **CONTRIBUTING.md**: rewritten to document the worktree-based development flow
